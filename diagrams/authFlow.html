Route guard checks: cookie present?

Login
1

2
signInWithPassword() → Supabase Auth
Supabase validates credentials. Returns pass or fail.
Invalid credentials
LoginAPI returns 401 to browser. Flow ends.
Valid credentials
Proceeds to look up user record in users table.
3
SELECT from users table

Not in users table

User found
setUserSession({"{"}id, email, org_id, role{"}"} → encrypted + signed cookie set on browser.
4
Route guard re-checks → redirect /app
Cookie now present. Guard passes. User lands on the submission inbox.

1
Browser makes any API request
Encrypted cookie sent automatically with every request to Nitro.
2
API route calls getSessionUser(event)
org.ts calls getUserSession(event) → decrypts cookie → returns {"{"}id, email, org_id, role{"}"}
No session / missing org_id
Returns 401 immediately. Request rejected.
Valid session
Proceeds with role-based data access.
Role-based data access
Applied to every database query
Underwriter
Query scoped to WHERE org_id=? AND user_id=? — own submissions only
Admin
Query scoped to WHERE org_id=? with optional ?userId= filter — all org submissions
Special cases
Admin-only routes Call requireAdmin() — throws 403 if role ≠ admin. Covers: guidelines upload, user list.
Email inbound — bypasses session entirely Org derived from the To: address handle. user_id = null. Admins only can see these submissions in the app.
Browser uses anon key for Realtime only All data fetching goes through Nitro API. Server uses service role key — full access. Browser never touches the database directly.
