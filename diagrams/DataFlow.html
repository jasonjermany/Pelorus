Entry points
A
Two ways a submission enters
File upload Underwriter uploads files or pastes text via modal → POST /api/submissions/ingest · validate session · read multipart

Email inbound Broker emails handle@mail.pelorusai.io → POST /api/email/inbound · extract handle → lookup org · verify sender non-empty · grab attachments or wrap body as .txt
B
INSERT submissions row
status: processing · raw_text: empty · user_id: session.id (upload) or null (email)
C
Return 200 immediately → setImmediate
Client and SendGrid get a fast response. Background job begins via setImmediate — all heavy work is async from here.
Document parsing
1
Send all files to Reducto AI in parallel
Each file parsed independently and concurrently. Returns typed chunks per file.
2
filterChunks()
Drop: length < 50 chars · header/footer-only blocks
Keep: Text · Table · List Item · Key Value · bullet chars regardless of block type
3
Label + concatenate → raw_text
Each file labeled with === DOCUMENT: filename.pdf === then all concatenated. UPDATE submission.raw_text in Supabase.
4
getGuidelineChunks(orgId)
Fetch ALL chunks ORDER BY chunk_index · fetch risk_profile_fields from organizations
Split: pinned (is_pinned=true → hard stops) and standard guidelines
Build: hardStopsText · guidelinesText (with page refs) · hardStopCheckList (one entry per pinned chunk)
Parallel Claude calls
3 calls fire simultaneously — Flags fires early off the Checks stream
Insights
Pattern recognition · market context · consistency check · coverage gap analysis
Sonnet 4.6 · 2k tokens
Risk profile
Extract risk_profile_fields — each with value + source citation (v5 data model)
Haiku 4.5 · 5k tokens
Checks stream (streaming ON)
Forced tool_use: submit_evaluation. Watches partial JSON for decision token (PROCEED|REFER|DECLINE). As soon as decision visible → fires Flags early. stream.finalMessage() → checksResult = decision + guideline_checks[]
Sonnet 4.6 · 6k tokens
Flags — early fire then re-run
Fires ahead with decision + empty checks to save latency. Re-runs after server enforces decision with full checksResult. Returns: flags · recommendation · favorable_factors · dimension_scores · composite_score
Haiku 4.5
Server enforcement + assembly
Hard enforcement rules — deterministic, overrides Claude
Any guideline_check FAIL → DECLINE  ·  Any guideline_check REVIEW → REFER  ·  Otherwise → PROCEED
Logs warning if Claude's decision differed from enforced result
→
Promise.all — await all remaining calls
Insights, Risk profile, and re-run Flags all resolve. Assembled into final verdict JSON:
decisionPROCEED · REFER · DECLINE
composite_score0–10 holistic judgment
guideline_checksper-rule pass/review/fail
recommendationnarrative next action
flagsrisk signal list
favorable_factorspositive signals
dimension_scoresper-axis 0–10
insightspattern + market analysis
missing_info2–5 priority gaps
risk_profileextracted fields + citations
