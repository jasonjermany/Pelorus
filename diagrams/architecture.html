What happens from the moment a submission arrives to when the underwriter sees a verdict.

Email inbound Broker emails handle@mail.pelorusai.io → SendGrid webhook → POST /api/email/inbound (extracts handle, looks up org, verifies sender non-empty)
B
INSERT submissions row
status: processing · raw_text: empty · user_id: session.id (upload) or null (email)
C
Return 200 immediately
Client and SendGrid get a fast response. All heavy work happens async via setImmediate.

All files parsed in parallel. Returns typed chunks: Text, Table, List Item, Key Value.
2
filterChunks()
Drop: length < 50, header/footer-only blocks. Keep: Text, Table, List Item, Key Value. Keep bullet chars even if mislabeled by Reducto.

Each file labeled: === DOCUMENT: filename.pdf === then all concatenated. UPDATE submission.raw_text.

Fires ahead with decision + empty checks to save time. Re-runs after server enforces decision with full checksResult. Returns: flags, recommendation, favorable_factors, dimension_scores, composite_score.

Guidelines upload flow

Guidelines are preprocessed once at upload time. Every subsequent submission just fetches the pre-processed chunks — no re-parsing on each run.

1
PDF uploaded via /app/settings
/api/guidelines/upload receives the carrier's guideline document.
2
Reducto parses the PDF
Returns structured text chunks from the guideline document.
3
Two Claude calls process chunks
extractHardStops() identifies hard stop rules → marked is_pinned=true. classifyChunks() filters cover pages, generates summaries for each chunk.
4
DELETE old + bulk INSERT new chunks
Re-uploading a guideline fully replaces it — no appending. Chunks stored in guideline_chunks scoped to the carrier's org. Extended cache TTL applied on Anthropic API calls.

Key architectural decisions
setImmediate for background work
API responds immediately, evaluation happens async. Keeps response times fast regardless of how long Claude takes to process.
Sonnet for reasoning, Haiku for extraction
Expensive guideline comparison and insights use Sonnet. Fast structured extraction (risk profile, flags) uses Haiku. Optimizes cost without sacrificing quality on the calls that matter.
Guidelines preprocessed at upload
guideline_chunks created once at upload. Every submission just fetches pre-processed chunks. No re-parsing on each run. Extended cache TTL further reduces cost.
Dynamic prompt assembly per submission
System prompt = Persona (static) + Universal variables (always) + line-specific sections (only for lines requested). Keeps token count down for single-line submissions.
Composite score is holistic, not an average
Claude renders a qualitative verdict (hard stops → 0–2, clean risk → 8–10) independent of dimension scores. One hard stop tanks the composite regardless of all other dimensions.
Server enforces decision — Claude doesn't have final say
Any FAIL guideline check overrides to DECLINE. Any REVIEW overrides to REFER. Claude's verdict is a suggestion; the enforcement logic is deterministic and auditable.
DELETE old + INSERT on guideline re-upload
Re-uploading a carrier guideline fully replaces chunks. No stale data, no append conflicts. Clean slate on every update.
Flags fires early off the Checks stream
As soon as the decision token is visible mid-stream, Flags fires with what it has. Re-runs with full data after enforcement. Saves latency on the most visible output.
